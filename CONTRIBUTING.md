# Contributing to Find Your Twin Project

First off, thanks for taking the time to contribute!

I welcome contributions from everyone, as it's a great way to share knowledge and improve the project. No matter if you are here to fix a bug, improve the documentation, or add a new feature, your help is appreciated.

## How to Contribute

1. Reporting Bugs

Found a bug?

- Please check if the issue has already been reported.
- Open an Issue on GitHub.
- Be clear and concise.
- Include steps to reproduce the bug so it can be fixed faster.

2. Suggesting Features

Have an idea to make the project better?

- Open an Issue and label it as a feature request.
- Explain why this feature would be useful and how it should work.

3. Submitting Code (Pull Requests)

Ready to write some code? Follow these steps:

- Fork the repository to your own GitHub account.
- Clone the project to your machine.
- Create a new branch for your work:

```bash
git checkout -b feature/my-new-feature
```

Make your changes. Keep your code clean and readable.
Test your changes to make sure the app still runs correctly.
Commit your changes with a clear message:

```bash
git commit -m "Add face alignment feature"
```

Push to your branch:

```bash
git push origin feature/my-new-feature
```

Open a Pull Request (PR) on the main repository. Describe what you changed and why.

## Coding Guidelines

- Follow the existing coding style.
- Add comments to explain complex logic.
- If you add a new library, make sure to update pyproject.toml.

## Code of Conduct

This project follows the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/1/code_of_conduct/). By participating, you are expected to uphold this code. Please report unacceptable behavior to the project owner.

Thank you for helping make this project better!
